# Copy / Microcopy System

> Per-style + per-industry copy patterns. Use these to generate non-AI-sounding text that matches the visual language. The most common AI tell is not visual — it's the copy.

---

## The AI Copy Blocklist

Never write these. They are the copy equivalent of pill buttons.

| Banned phrase | Why it's AI | What to write instead |
|--------------|-------------|----------------------|
| `✨ Introducing [Product]` | Every v0/Lovable launch | `[Product] is live.` or just the product name |
| `Get started for free` | 100% AI SaaS default | `Start free` / `Open an account` / `Try it now` |
| `Trusted by X+ teams` | AI social proof boilerplate | Show a real client name or nothing |
| `Everything you need to [verb]` | Features section H2 template | Name the actual thing: `The billing stack for...` |
| `Simple, powerful, [adjective]` | 3-adjective hero pattern | One concrete claim |
| `We're on a mission to...` | About page AI cliché | Start with a fact about the world |
| `Loved by developers` | Testimonial section filler | Quote an actual person, name + title |
| `Level up your [X]` | Gaming metaphor overuse | Say what it actually does |
| `Built for [noun]s` | Narrowing claim without substance | Say the specific pain it solves |
| `[Feature] that just works` | Implicit comparison to nothing | Say what it replaces or improves |

## Per-Style Headline Formulas

### Swiss Editorial

**Principle:** Typographic assertion. No puns. No verbs that explain. State a fact and let the form carry the weight.

```
Formula: [Noun phrase] — [qualifying noun phrase]
Formula: [Number] [noun]
Formula: [Single declarative sentence, period]

Examples:
  "Publishing infrastructure."
  "01 — Editorial CMS"
  "The archive is the product."
  "Type sets the pace."
  "Designed in Zürich. Shipped worldwide."
```

**CTA verbs:** `Open account` / `Read the docs` / `Subscribe` / `See all work` / `Request access`

**Forbidden:** exclamation marks, emoji, `✨`, `🚀`, `→` as decoration (use as semantic arrow only)

---

### Magazine Longform

**Principle:** Editor's voice. The headline is the story hook — makes the reader need to know what comes next.

```
Formula: [Provocative question]
Formula: [Unexpected contrast]: [consequence]
Formula: The [noun] that [surprising verb phrase]

Examples:
  "Why your pricing page is lying to you"
  "Small teams, outsized impact: the new design ops"
  "The font that stopped a war (and other typographic myths)"
  "Against the scroll."
```

**CTA verbs:** `Read more` / `Subscribe to the dispatch` / `Explore the archive` / `Join the conversation`

**Forbidden:** `Get started`, `Try for free`, any button in a hero

---

### Fashion / Luxury

**Principle:** Absence speaks. Short, atmospheric. No explanation — the name is enough.

```
Formula: [Season] [Year]
Formula: [Single noun]
Formula: [Material / feeling noun]
Formula: [French/Italian word if authentic to brand]

Examples:
  "Spring 2026"
  "Permanence."
  "Made to be worn."
  "The Silk Edit"
  "Quiet luxury."
```

**CTA verbs:** `Discover` / `Explore the collection` / `View lookbook` / `Enter`

**Forbidden:** any price mention in hero, `Shop now`, exclamation marks, `Free shipping`

---

### Brutalist

**Principle:** Direct to the point of rudeness. No softening. No promises. Facts only.

```
Formula: [All-caps declarative]
Formula: [Verb + object, period]
Formula: [Blunt question]
Formula: [Number + noun]

Examples:
  "WE MAKE WEBSITES."
  "This is an agency."
  "Read this first."
  "12 clients. No BS."
  "Work for hire."
```

**CTA verbs:** `Hire us` / `Read` / `Contact` / `Open` (no decoration)

**Forbidden:** `✨`, emoji, any qualifier ("perhaps", "maybe", "we think"), subheadline that repeats the headline

---

### Type-First Monochrome

**Principle:** Typography is the image. The headline does all visual work. Often poetic, abstract, architectural.

```
Formula: [Fragmented phrase / typographic poem]
Formula: [One word, large]
Formula: [System/structure language]

Examples:
  "Space between words."
  "Structure"
  "The grid holds everything."
  "Measure. Set. Print."
  "Nothing decorative."
```

**CTA verbs:** `View work` / `Read` / `Collaborate` / `Index`

**Forbidden:** Any image reference in copy, metaphors involving color, emoji

---

### Neobrutalism

**Principle:** Loud, honest, opinionated. Product-forward. Numbers preferred over adjectives.

```
Formula: [Bold product claim with number]
Formula: [Comparison that names the winner]
Formula: [Feature as headline]

Examples:
  "Ship 3x faster than Notion."
  "Your Figma plugin is dead. Use this."
  "47,000 developers. One tool."
  "No more copy-paste. Ever."
```

**CTA verbs:** `Try free` / `Get the app` / `Install now` / `Start building`

**Forbidden:** `✨`, anything "magical", corporate softening

---

### Corporate Clean

**Principle:** Confident, clear, business-value focused. Numbers and outcomes over features.

```
Formula: [Outcome] for [audience]
Formula: [Business metric] + [product category]
Formula: [Verb] [business outcome] with [product]

Examples:
  "Enterprise analytics for modern finance teams"
  "Cut review cycles from weeks to days"
  "The compliance platform your legal team trusts"
  "Revenue reporting that closes books on time"
```

**CTA verbs:** `Request a demo` / `Talk to sales` / `Start a trial` / `See pricing`

**Forbidden:** startup-speak, `✨`, "magic", casual tone

---

### Editorial Newspaper

**Principle:** Informational, journalistic. Headline + deck. No marketing language.

```
Formula: [News headline style] — present tense, active voice
Formula: [What happened] + [why it matters]

Examples:
  "Design agencies rethink remote after 2024 report"
  "Inside the studio: how Bureau Borsche builds identity systems"
  "The case against components"
```

**CTA verbs:** `Read` / `Subscribe` / `Continue reading`

**Forbidden:** any promotional language, `Explore`, `Discover`, `Journey`

---

### Aurora UI / Glass / Claymorphism

**Principle:** Aspirational, feature-forward, slightly playful. AI-adjacent products live here — but avoid clichés.

```
Formula: [Product + power verb + outcome]
Formula: [Future-state claim]
Formula: [AI claim without "AI magic"]

Examples:
  "Your design system, generated."
  "From brief to shipped in one afternoon."
  "Everything connected. Nothing lost."
  "Build faster than you can sketch."
```

**CTA verbs:** `Start building` / `Get early access` / `Join the waitlist` / `Try for free`

**Forbidden:** `Harness the power of AI`, `AI-powered ✨`, `Magic`, `Transformative`

---

### Kinetic / Type-First (Motion)

**Principle:** The copy and the motion are inseparable. Short phrases designed to animate. Each word earns its place.

```
Formula: [2-4 words, animates letter by letter]
Formula: [Verb phrase that carries kinetic energy]

Examples:
  "Move with purpose."
  "Speed is design."
  "Every second counts."
  "Make it move."
```

---

### Minimal Float

**Principle:** Soft, confident, human. Not startup-loud, not enterprise-stiff.

```
Formula: [Human observation about the problem]
Formula: [Warm product positioning]

Examples:
  "Good work starts with good tools."
  "Design shouldn't feel like a battle."
  "Your best thinking, organized."
  "Made for the way you actually work."
```

**CTA verbs:** `Get started` (allowed here) / `Try free` / `Learn more` / `Sign up`

## Per-Industry Copy Patterns

### SaaS

**Hero headline formula options:**
```
Option A: [Verb] [outcome] — [qualifier]
  → "Ship products customers love — without the bottleneck"

Option B: [Product category], [redefined]
  → "Project management, finally honest"

Option C: [Number] [time unit] to [outcome]
  → "30 minutes to your first deployment"

Option D: [Audience] [verb phrase]
  → "Design teams ship on Friday"
```

**Subheadline (1 sentence max):**
```
[How + product name + solves the pain]
→ "Linear connects roadmaps to actual work, so sprint planning isn't theater."
→ "Render auto-deploys every branch so staging is never stale."
```

**Features section copy pattern:**
```
H2: NOT "Everything you need" — use specific:
  → "The [noun] stack" / "How [product] handles [job to be done]"

Each feature: [Verb phrase] + [consequence]
  → "Merge without asking — zero-config branch deploys"
  → "Review in context — comments live on the component, not in Slack"
```

**Pricing tier names (avoid "Starter / Pro / Enterprise"):**
```
Option A: Usage-based names → "Solo / Studio / Scale"
Option B: Identity names → "Free / Builder / Team"
Option C: Swiss approach → "01 Free / 02 Pro / 03 Enterprise"
Option D: Brutalist → "YOU / TEAM / CORP"
Option E: Minimal → just the price, no tier name
```

**Empty state copy:**
```
Pattern: [What will live here] + [single CTA, no pressure]
  → "No projects yet. Create your first →"
  → "Nothing shipped today. Keep going."
  → "Your pipeline is clear."
```

**Error messages:**
```
Bad:  "Something went wrong. Please try again."
Good: "We couldn't save that — the session timed out. [Reload page]"

Bad:  "Invalid input."
Good: "Email addresses look like 'you@company.com'."

Bad:  "Permission denied."
Good: "This project is invite-only. Ask your admin to add you."
```

### E-Commerce

**Product card:**
```
Name: [Descriptive product name, no SEO stuffing]
  → "Merino Crew — Natural White"  (not "Premium Merino Wool Crew Neck Sweater White")

Price display: Just the number + currency. No "Only $49!"
Availability: "3 left" not "Hurry! Only 3 left in stock!"
```

**PDP (Product Detail Page) copy:**
```
H1: [Product name]
Subhead: [1 line material or key differentiator]
Description: 2-4 sentences max, factual, sensory.

Bad:  "Experience the ultimate in comfort with our premium..."
Good: "250gsm organic terry. Pre-washed. Ships flat to reduce carbon."
```

**Cart copy:**
```
CTA: "Go to checkout" not "Proceed to checkout" (bureaucratic)
Empty cart: "Nothing here yet." not "Your cart is empty. Start shopping!"
Success: "Added." or "In your bag." (brief, calm)
```

**Fashion-specific copy:**
```
Avoid: adjectives that describe what users should feel
Use: materials, process, origin, number of items
  Bad:  "Luxuriously soft cashmere for the modern woman"
  Good: "Grade A cashmere. 180gsm. Made in Inner Mongolia."
```

### Editorial / Blog

**Article titles:**
```
Pattern A: [Action noun phrase]  → "Against simplicity"
Pattern B: [Statement]           → "The web is broken by design"
Pattern C: [Question]            → "Who killed the long-form web?"
Pattern D: [Colon structure]     → "Layout theory: why grids lie"
```

**Author byline patterns:**
```
Swiss/Minimal: "By [First Last]" — nothing else
Magazine: "Words by [Name] — Photography [Name]"
Brutalist: "[NAME]" — all caps, no "by"
Editorial: "Reporter: [Name]" or "[Name] reports from [city]"
```

**Pull quote selection rule:**
Choose the most surprising claim in the article, not the most comfortable one.
```
Bad pull quote:  "Design is important to any product."
Good pull quote: "The best interface I've ever built was never shipped."
```

---

### Portfolio

**About page opener (avoid the bio-cliché):**
```
Bad:  "I'm a designer passionate about creating meaningful digital experiences."
Good: Start with a project, an opinion, or a fact.
  → "I've redesigned the same landing page 47 times. Each time I learn something new about attention."
  → "Based in Amsterdam. 12 years in type. Currently: Pentagram NY."
  → "I think most SaaS products look the same. This is what I'm doing about it."
```

**Project card (case study preview):**
```
Bad:  "UI/UX design for a fintech startup"
Good: "[Client name] — [what changed]"
  → "Monzo — redesigned the card spend view for 8M users"
  → "Unseen — identity for a gallery with no budget and too many opinions"
```

**Work section headline:**
```
Avoid: "My Work" / "Projects" / "Case Studies"
Use: "Work" / "Selected projects" / Or just year ranges: "2022–2026"
Swiss: Just a number or index
Magazine: "The portfolio"
```

---

### Docs

**Conceptual intro (the "what is this?" paragraph):**
```
Bad:  "Welcome to our documentation. Here you'll find everything you need."
Good: Start with the primary action a user wants to accomplish.
  → "Deploy your first app in 5 minutes. Start with [Getting started →]."
  → "The API has two parts: a REST interface and a real-time subscription layer."
```

**Code block label pattern:**
```
Minimal: Just the language (bash, tsx, json)
With context: "Install" / "Usage" / "Example" (no exclamation marks)
Never: "🎉 Example code" / "Super simple!" 
```

**Callout / admonition copy:**
```
NOTE: For clarifying information (not urgent)
WARNING: For destructive or irreversible actions
TIP: For optimization (not required, but faster)
Never: "⚡ Pro tip:", "💡 Did you know:", emoji-led callouts
```

## Button Label Library

### Action buttons

| Context | Bad (AI default) | Good alternatives |
|---------|-----------------|-------------------|
| Primary signup | `Get started for free ✨` | `Create account` / `Start free` / `Sign up` |
| Product try | `Try it for free` | `Open the app` / `Start building` / `Try free` |
| Demo request | `Book a demo` | `Talk to us` / `Request a demo` / `See it live` |
| Learn more | `Learn more →` | `See how it works` / `Read the docs` / `Explore` |
| Pricing | `See pricing` | `View pricing` / `Compare plans` |
| Contact | `Get in touch` | `Contact` / `Reach out` / `Email us` |

### State labels

| State | Bad | Good |
|-------|-----|------|
| Loading | `Loading...` | `Saving…` / `Processing…` / `Connecting…` |
| Success | `Done! ✅` | `Saved` / `Sent` / `Deployed` |
| Error | `Error!` | `Couldn't save` / `Connection failed` |
| Empty | `No results found 😞` | `Nothing here yet` / `No results` |
| Disabled | `Unavailable` | `Upgrade to use this` / `Admin only` |

### Navigation labels

| Pattern | Bad | Good |
|---------|-----|------|
| Back | `← Go back` | `← Back` or just `←` |
| Next | `Next step →` | `Continue →` |
| Submit form | `Submit` | Use action verb: `Save changes` / `Send message` / `Create project` |
| Cancel | `Cancel` | Use opposite of confirm: `Discard` / `Never mind` / `Keep editing` |

## Microcopy Templates

### Onboarding

```
Step indicator: "2 of 4" not "Step 2/4: Set up your workspace"
Skip link: "Do this later" not "Skip for now ✨"
Progress: "Almost done — just your team size" not "You're almost there! 🎉"
```

### Empty states by page type

```
Dashboard (new user):
  "Your dashboard is empty. [Create first project] or [import existing work]."

Search (no results):
  "No results for '[query]'. Try broader terms or [browse all]."

Notifications (none):
  "You're up to date." (not "No notifications yet!")

Activity feed (no activity):
  "Nothing yet." (not "No activity to show 😶")

Inbox (empty):
  "Inbox zero." (not "Your inbox is empty. Great job!")
```

### Form validation

```
Required field:     "[Label] is required"        (not "This field is required")
Format error:       "Enter a valid email"         (not "Invalid email address")
Too short:          "Password must be 8+ characters"
Too long:           "Keep it under 200 characters ([n] left)"
Mismatch:           "Passwords don't match"
Already taken:      "That username is taken. Try [suggestion]?"
Network error:      "Couldn't submit — check your connection and try again"
```

### Confirmation dialogs

```
Delete:
  Title: "Delete [item name]?"
  Body:  "This can't be undone."
  CTA:   "Delete" (red) + "Cancel"
  (Not: "Are you sure you want to permanently delete this item? This action cannot be reversed." + "Yes, delete it!" + "No, go back")

Destructive action:
  Require typing the item name for high-stakes deletes
  Body: "Type [item-name] to confirm:"
```

### Tooltips

```
Rule: Only add if the label alone isn't enough. One sentence max.
Bad:  "Click this button to save your changes to the database"
Good: "Save (⌘S)"
Good: "Changes are auto-saved every 30 seconds"
```

## Anti-AI Copy Checklist

Before finalizing copy, check:

- [ ] No `✨` emoji in product copy (acceptable only in social/informal contexts)
- [ ] No "Get started for free" CTA
- [ ] No "Everything you need to [verb]" headline
- [ ] No 3-adjective list ("simple, powerful, flexible")
- [ ] No "We're on a mission to [broad aspiration]"
- [ ] No "Loved by [role]s" social proof header
- [ ] Hero headline ≤ 8 words (if following Swiss/Type-First/Brutalist)
- [ ] Pricing tiers are NOT named Starter/Pro/Enterprise
- [ ] Error messages say what happened and what to do — not just "Error"
- [ ] Empty states are calm, not cheerleading ("Nothing here" not "Wow, so empty! 😅")
- [ ] Button labels are verbs, not "Learn more" or "Click here"
- [ ] No passive voice in feature descriptions
